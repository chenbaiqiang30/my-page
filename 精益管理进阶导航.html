MODULE 01 初期清扫：先把“看不见的问题”，变成“看得见的清单”

很多工厂一上来就谈 TPM、谈自动化、谈数字化，结果连设备基础状况都说不清。 初期清扫的价值，不在“扫得多干净”，而在于： 把所有异常一口气翻出来，形成第一版设备问题数据库。

4~6 周集中体检 “跑通第一个闭环”的最佳切入口
一、初期清扫的四个真正目标
不是搞卫生，而是建一套“设备健康档案”。
›
4 个目标
  • 看得见：让所有漏油、渗水、积粉、异响统统暴露出来。
  • 说得清：每一个异常点，都有照片、编号、位置说明。
  • 可统计：按设备/工序整理出“高发区域”和“重复问题”。
  • 可追踪：为后面的点检标准、QC Story 提供种子问题库。
3 张必备表
  • 《设备初期清扫检查表》：今天要扫哪几台、谁负责、结果如何。
  • 《问题点一览表》：异常编号 / 设备 / 位置 / 现象 / 初步判断。
  • 《改善候选清单》：按安全 ＞ 环保 ＞ 影响停机排序的改善优先级。

你可以把这一步理解为“体检 + 建档”。如果初期清扫做得扎实， 后面的点检、自主保全就有了真正的“靶子”，而不是靠感觉拍脑袋。

二、现场推进的三个“不要”与三个“一定要”
初期清扫最容易做成运动式，关键在于管理者怎么“收口”。
›
3 个不要
  • 不要由管理干部代劳，替员工把设备收拾得漂漂亮亮。
  • 不要一开始就追求“全工厂一次搞定”，导致虎头蛇尾。
  • 不要把清扫当“临时检查应付上面”，扫完就结束。
3 个一定要
  • 一定要让操作工亲自参与，让他成为设备“第一责任人”。
  • 一定要把发现的问题拍照、贴标、记录，留下证据链。
  • 一定要在结束时开一次“问题展览会”，让大家一起看数据说话。

指标不是“扫了多少灰”，而是：我们新增了多少条有价值的问题点。 这才是后续精益改善的“金矿”。

MODULE 02 点检标准：把“师傅脑子里的经验”变成“谁都看得懂的表”

清扫只是把病找出来，点检标准才是把“怎么防止再犯”写清楚。 真正有用的点检表，是一线愿意用、会用、能用的 “防错脚本”。

从一次性清扫 → 日常点检 经验写入纸面，防止“靠记忆管理”
一、用“五个优先”选出要写进点检表的项目
点检表不是越多越好，而是越“关键”越好。
›
5 个优先级
  • 优先 1：与人身安全相关的部位。
  • 优先 2：涉及环保/法规风险的部位。
  • 优先 3：一旦失效会造成停机或大量报废的关键点。
  • 优先 4：初期清扫中被反复发现的“顽固问题点”。
  • 优先 5：员工主观认为“总是搞不清楚”的薄弱点。
从“看不懂”到“秒懂”的三要素
  • 有图：用照片/示意图标出“检查位置在哪里”。
  • 有标准：清楚写明“什么状态算 OK / 不 OK”。
  • 有动作：出问题后“第一时间应该做什么、通知谁”。

一张好的点检表，你可以交给刚入职 1 个月的新人，他不问你也能照着做完， 这才叫做“把经验放进系统里”。

二、从“纸面点检”到“习惯点检”的闭环
很多厂有表没人填，有记录没人看——缺的是这一条闭环链路。
›
6 步闭环链路
  • ① 发现点：来自初期清扫 / 故障记录 / 员工提案。
  • ② 分析点：判断是一次性问题，还是需要标准化的共性问题。
  • ③ 写标准：把动作、频次、责任人写进点检表。
  • ④ 教员工：通过 OJT 实操训练，而不是 PPT 宣讲。
  • ⑤ 看执行：班组长巡线时随机抽查点检表与现场是否一致。
  • ⑥ 再优化：每季度把“多次出问题的点”升级版标准。
典型坑位
  • 把点检表当“ISO 证据”，不当现场工具。
  • 只追求项目多，不考虑员工每天根本做不完。
  • 从不根据故障反馈更新点检表，导致表越来越失真。

你可以设一个简单 KPI：“一年修改过几次点检表”。 如果答案是 0，说明点检表只是挂在墙上的装饰品。

MODULE 03 管理者心态：设备保全，最后都要落在“怎么带人”上

PPT 上写的是一句话：“设备保全，最终是人的课题。” 真正的意思是：设备出问题时，你先怪人，还是先怪系统， 决定了这家公司能走多远。

班组长 / 科长必修心法 从“盯人”到“盯系统”的转弯点
一、设备出故障时，优秀管理者先问的三句话
相信：操作工有能力做设备的基础医生——前提是你给了他正确的工具。
›
1. 基准是否清晰？（标准责任）
  • 有没有把“怎么保养”写进点检表，而不是一句“注意保养”。
  • 员工是不是能通过目视看板知道：今天、这一班要做到什么水平。
  • 同一个设备，不同班次是否做到同一标准，还是各看各的心情。
2. 培训是否到位？（能力责任）
  • 新员工有没有“跟岗+实操”记录，而不是签个名就算通过。
  • 有没有人负责把师傅经验转成教材，而不是“嘴传嘴”。
  • 每次典型故障，是否变成下一次培训的真实案例。
3. 自主保全活动是否坚持？（习惯责任）
  • 是“搞一阵子活动”，还是已经变成每班必做动作。
  • 班组长巡线时，是只看产量，还是同时看清扫 / 点检状态。
  • 高层的日 / 周 / 月例会上，有没有固定看“设备 × 人 × 结果”的三张图。
EXPERT VIEW · 三层责任模型

当设备出故障时，可以用这三层快速自检：

  • 结果责任：设备停了，这是谁的 KPI？（通常是主管）
  • 过程责任：点检记录有没有、有没有按频次做？
  • 能力责任：员工有没有被教会“怎么看、怎么判断、怎么应对”。

把所有问题都简单归结为“员工不负责”，是最廉价的管理方式； 真正高手，会先把镜子照向标准 / 培训 / 机制—— 也就是你自己。

你可以每次故障后开一个 10 分钟的小复盘，只问三句： “标准有没有？教过没有？检查过没有？” 三个问题走完，再谈员工态度，才公平。

二、从“盯人”到“盯系统”的四个心态转变
设备好不好，其实是在放大管理者的心态和习惯。
›
4 个转变
  • 从“出了问题找替罪羊” → “出了问题找机制漏洞”。
  • 从“事后救火” → “事前预防 + 早期预警”。
  • 从“我一个人盯设备” → “让每个员工都成为设备的主人”。
  • 从“短期指标至上” → “长期体质优先（安全·稳定·效率）”。
一条很残酷但真实的判断

如果一个车间的设备总是脏、总是漏、总是坏，真正被放大的其实不是员工的问题， 而是管理者对人的看法：
你是把员工当成“能成长的伙伴”，还是“只会犯错的成本”。

你可以每个月给自己打一次分：
「这 30 天里，我是更多在抱怨人，还是更多在修系统？」
这就是“管理者心态曲线”，决定了你团队的天花板。

MODULE 04 员工能力 & 自主保全：设备的“家庭医生”在一线

真正健康的工厂，不是设备科一个部门在拼命，而是每位操作工都具备基础“诊断 + 保养”能力。

技能矩阵 / OJT / 自主保全等级 从“会开机器”升级到“会照顾机器”
一、员工能力的“四级台阶”
不要笼统说“员工能力不行”，要看他现在在第几级台阶。
›
4 级能力定义
  • L1：会用——能按作业指导书安全操作设备。
  • L2：会保养——能按点检表完成日常清扫、加油、紧固。
  • L3：会判断——能分辨“正常/异常”，并给出初步原因。
  • L4：会教人——能把以上三点教给新人，带出徒弟。
一张简单但很有用的矩阵

把每个员工的设备能力做成一张技能矩阵， 标记出谁是 L1、谁是 L2、谁能做 L3/L4。
你会立刻发现：有没有人真正承担“家庭医生”的角色。

自主保全的本质，不是“多干一点活”，而是让员工在“照顾设备”的过程中， 建立起专业感和成就感——这会直接反映在品质与效率上。

二、让自主保全真的“活起来”的三个关键动作
很多地方自主保全做不动，不是员工不愿意，而是缺这三样东西。
›
3 个关键动作
  • 给员工一个专属的“设备名片”——写清楚这台设备的关键点。
  • 设立“今日发现亮点”——哪怕是发现一处小渗漏也要被看见。
  • 每月评一次“最佳设备守护者”——奖励的是习惯，不是一次性的成绩。
你可以马上做的小改动
  • 把“清扫/点检/加油”从杂事，改名叫“设备守护动作”。
  • 在早会固定 3 分钟，让员工轮流分享一个设备小发现。
  • 把优秀的自保案例做成 A4 卡片贴在设备旁，给人也给设备“贴标签”。

当员工开始主动告诉你：“这个点我怎么看着怪怪的”， 说明你的自主保全已经从“任务”变成了“眼睛”。

MODULE 05 EXPERT VIEW：设备 & 人 & 经营体质，是同一套系统的三个界面

胡光书那句“通过改善设备体质，最终改善经营体质”， 不是一句漂亮话，而是一条可以落在报表上的因果链。

从单点改善 → 体质升级 每年四个精益闭环
一、设备 · 人 · 经营的“三条线”怎么串起来
不要只看 OEE，也要看“背后的人”和“最后的利润”。
›
三条线
  • 设备线：故障率 / OEE / 计划外停机时间。
  • 人线：技能矩阵成熟度 / 自主保全参与率 / 提案数量。
  • 经营线：单位产量成本 / 交付稳定性 / 客户投诉率。

一家工厂如果只盯着“产量和成本”，不看设备和人， 很容易把体质透支掉；反过来，只谈文化不看数字，也走不远。 真正的高手，会让这三条线互相解释、互相支撑。

一年四个闭环的示例节奏
  • Q1：初期清扫 + 问题数据库建立（打基础）。
  • Q2：重点设备点检标准完成 + 试运行（固标准）。
  • Q3：自主保全全面展开 + 班组评比（养习惯）。
  • Q4：把 OEE / 成本 / 交付等经营指标，与前面三步挂钩复盘（看体质）。

这样做的好处是：每个季度都有一个清晰主题， 既不会“运动式一阵风”，也不会让员工觉得精益是空的。

二、把这一页真正变成“企业版”导航图
不是收藏在电脑里，而是要活在会议室和现场。
›
建议使用方式
  • 年度经营会：用这五个模块做精益主题报告的大纲。
  • 月度例会：每月只挑一个模块，讲一个真实案例。
  • 新任干部培训：让每位班组长结合自己设备做一份“本车间版本”。
给自己留的两个问题
  • 如果只保留一个模块在今年重点做，你会选哪一个？为什么？
  • 一年后再回到这页时，你希望看到哪些数字已经发生了变化？

这两个问题，就是你对自己的“经营体质期望值”。 建议写在纸上、拍照，明年的你会感谢现在认真的自己。
